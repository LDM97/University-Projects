How the Internet Works

The internet is a fairly complex digital structure that links the world together. When making a website for it, it is invaluable to know just what it is that you are working with. The sections below will introduce you to the basics of how the internet works in order to give you this deeper understanding to help you make your websites better.

Types of Network

The internet essentially connects computers together. This can be done in several ways, though two of the main ones are, peer-to-peer and client / server. Client / server works by the client (E.G. a user on their home computer) connecting to a remote server which will supply the information to display the given web page. Peer-to-peer uses no central server to store information, and connects the clients together directly. Essentialy, each client's computer works like a server, storing, sending and recieving the files to and from their peers (other clients).

Usually, you will be developing web pages for a client / server setup, where your page will be stored on the server. Making it easy to access and change / update the page if and when it is required.

Protocols

Protocols define set standards for passing data between different devices. Without these, it would be impossible for all types of devices to connect to each other. Devices need to agree on a set standard otherwise inter device communication becomes impossible, as data for one device would be unreadable to another. Below are some of the many protocols used to standardise the internet. There are many more than these, but these are the most basic / essential ones.

Protocols
Protocol Description
HTTP (HyperText Transfer Protocol) Main protocol for web transactions.
FTP (File Transfer Protocol) Used to standardise filex exchange between computers.
SSL (Secure Sockets Layer) Used for data encryption.
TCP (Transmission Control Protocol) Handles direct connection between client / server software and network interfaces.
IP (Internet Protocol) Deals with delivery of packets to / from any device with a unique numeric identified (IP Address).

Accessing & Sending Data

Data is sent across the internet using packets. Packets contain a header and a body. The header contains addressing data (such as the IP address) to say where to send the data. The body contains the actual data itself, the web page, video, picture etc. IP routes packets using the information in the header to direct the information to their destination.

Data is, usually, accessed using a domain name. Other devices and servers are identified using an IP address, but these numeric addresses are not user friendly for a person to remember. The way to solve this is using domain names. An IP address is linked to a name (such as the Google search engine's IP address is linked to www.google.co.uk). This allows the user to type in the name rather than having to remember the numerical IP address. The links between names and IP addresses are stored on a Domain Name Server (DNS server). There are thousands of these across the world and when you type in a web address your computer connects to one of these DNS servers, gets the IP address and then connects to the destination using that.

Together packets and DNSs make the main backbone of the internet, allowing data to be sent and accessed.
